# GT Table Block

A WordPress plugin that enhances the core Table block with four features delivered
as a block variation with composable toggles.

## Features

| Feature | PR | Status |
|---|---|---|
| Sticky header row | 1 | ✅ Complete |
| Styled first column (header column) | 2 | 🔲 Stub ready |
| Sticky first column | 3 | 🔲 Stub ready |
| Merge / unmerge cells | 4 | 🔲 Planned |

All features are independent toggles — any combination can be active simultaneously.
See [PLAN.md](./PLAN.md) for architecture decisions, prior art, and known gotchas.

## Development

**Requirements:** Node 20+, npm 10+

```bash
npm install
npm run start   # watch mode
npm run build   # production build
```

The plugin uses [`@wordpress/scripts`](https://developer.wordpress.org/block-editor/reference-guides/packages/packages-scripts/)
with its default webpack config. No custom `webpack.config.js` is needed.

## File Structure

```
gt-table-block.php             Main plugin file — enqueues editor + frontend assets
src/
  index.js                     Entry point — imports all filters and styles
  variation.js                 registerBlockVariation('core/table', ...)
  filters/
    extend-attributes.js       Adds custom attrs to core/table via blocks.registerBlockType
    with-inspector-controls.js Injects InspectorControls panel via editor.BlockEdit
    with-extra-props.js        Applies CSS classes + inline styles in editor via editor.BlockListBlock
    with-save-element.js       Modifies saved HTML via blocks.getSaveElement (PR 3, PR 4)
  controls/
    StickyHeaderControls.js    PR 1 — toggle + offset field
    HeaderColumnControls.js    PR 2 — stub
    StickyColumnControls.js    PR 3 — stub
  style.scss                   Frontend + editor styles (all features)
  editor.scss                  Editor-only overrides
build/                         Generated by wp-scripts (git-ignored)
```

## Why a variation, not block styles?

Block styles (`registerBlockStyle`) are mutually exclusive — the editor presents
them as a radio group. A table needing both a sticky header *and* a styled first
column would be impossible. The variation with independent boolean attributes
solves this; every feature is a separate checkbox. See PLAN.md for the full
reasoning.

## Prior Art

This plugin deliberately extends `core/table` rather than replacing it.
Tableberg, PublishPress Blocks (Advanced Table), and Getwid all ship their own
custom table blocks. This plugin takes a different approach: a thin variation layer
that stays compatible with core as it evolves. See PLAN.md → Prior Art Consulted.
